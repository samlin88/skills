## 什么是Trae Skill？

先来说说什么是Skill吧。

其实很简单，Skill就像是给AI的"使用说明书"，
告诉AI在什么情况下应该做什么、怎么做。

比如你经常要写公众号文章，
就可以创建一个"公众号文章生成器"Skill，
以后只要说"帮我写篇公众号文章"，
AI就会自动按照你设定的流程来帮你生成。

**Skill能用来做什么？**
- 定义特定领域的工作流程
- 保存常用的提示词和模板
- 让AI更好地理解你的需求
- 打造专属的AI助手

## 手把手创建Skill（核心步骤）

好，我们开始！
就以我刚才创建的"微信公众号文章生成器"为例，
一步步教你怎么创建自己的Skill。

### 步骤1：确定Skill的功能和定位

首先得想清楚：
- 这个Skill是用来做什么的？
- 目标用户是谁？
- 核心功能有哪些？

比如我要做公众号文章生成器，
那核心功能就是：选题、大纲、正文、模板匹配、去AI味优化、联网搜索。

讲真，这一步我纠结了好久，
一开始想加很多功能，
后来发现贪多嚼不烂，
还是先把核心功能做好最重要。

给你们个建议：
一开始不要追求完美，
先做一个最小可用版本，
后面再慢慢迭代。

**Trae技巧**：如果你不知道Skill的功能，也没问题，可以用以下提示词：
```markdown
/plan 我想写一个skill，关于 微信公众号热门文章生成器skill，不知如何下手，请引导我，并做相关计划
```


### 步骤2：创建目录结构

打开终端，执行这行命令：
```bash
mkdir -p .trae/skills/wechat-article-generator
```

或者在文件管理器里手动创建文件夹也行，
路径要对：`.trae/skills/你的skill名字/`

说实话，我第一次创建的时候，
路径写错了，
结果AI找不到Skill，
折腾了半天，
后来才发现是路径的问题...
哎呀，现在想想还觉得好笑。

### 步骤3：编写SKILL.md主文件

这是最关键的一步！
在刚创建的目录里新建一个`SKILL.md`文件。

**frontmatter部分（必须有）：**
```markdown
---
name: "wechat-article-generator"
description: "微信公众号热门文章生成器，支持选题、大纲、正文生成、模板匹配、去AI味优化和联网搜索。Invoke when user needs to generate WeChat official account articles."
---
```

注意！`description`很重要，
一定要包含：
1. 这个Skill做什么
2. 什么时候调用它

讲真，我第一次写description的时候，
只写了做什么，
没写什么时候调用，
结果AI根本不知道什么时候用这个Skill，
后来看了文档才知道要加上"Invoke when..."这部分。

**正文部分（详细说明）：**
然后在下面写详细的使用说明，
比如：
- 功能介绍
- 使用方法
- 内置模板
- 示例对话
- 注意事项

这些内容写得越详细，
AI用起来就越顺手！

### 步骤4：补充详细文档（可选但推荐）

如果你的Skill比较复杂，
可以再创建一些辅助文档，
比如我就创建了：
- `workflow.md` - 工作流程详解
- `templates.md` - 文章模板系统
- `de-ai-optimization.md` - 去AI味优化策略
- `web-search.md` - 联网搜索集成方案
- `enhanced-features.md` - 增强功能详解
- `author-persona.md` - 作者人设
- `example.md` - 完整使用示例

这些文档AI都会自动参考，
能让Skill的表现更好！

讲真，一开始我觉得没必要写这么多文档，
后来发现写得越详细，
AI的表现就越好，
花的时间还是值得的！

# 微信公众号热门文章生成器

## 功能介绍

这是一个专业的微信公众号文章生成助手，特别适合干货/教程类文章创作。

### 核心功能

1. **选题助手** - 根据领域/关键词生成热门选题
2. **大纲生成** - 自动创建文章结构框架
3. **正文写作** - 基于大纲生成完整文章
4. **模板匹配** - 自动选择适合的文章模板
5. **去AI味优化** - 让文章更像真人写作
6. **联网搜索** - 获取最新资料和数据
7. **配图建议** - 推荐合适的配图方向

### 增强功能

8. **文件保存** - 将生成的文章保存为markdown文件
9. **多格式导出** - 支持markdown、纯文本等格式
10. **版本管理** - 保存多个版本，支持对比和回退
11. **SEO优化建议** - 提供关键词、摘要等优化建议
12. **标题A/B测试** - 生成多个标题，分析点击率潜力
13. **字数统计** - 自动计算字数和预估阅读时长
14. **文章迭代** - 支持逐步修改和优化文章

### 特殊要求（重要！）

15. **固定开头** - 所有文章开头必须是：【大家好，我是快乐IT啊。】
16. **大量个人经验** - 文章中必须加入丰富的个人经历、感受、踩坑记录等
17. **作者人设** - "快乐IT啊"是一个有10年IT经验的程序员，喜欢分享，说话幽默风趣

## 使用方法

### 快速开始

1. 告诉AI你想写什么主题的文章
2. 选择或让AI自动匹配合适的模板
3. 按流程生成：选题 → 大纲 → 正文 → 优化
4. 完成后自动保存为markdown文件

### 文章模板

内置干货/教程类专用模板：

- **问题解决型**：固定开头 → 痛点 → 方案 → 步骤 → 总结
- **知识科普型**：固定开头 → 概念 → 原理 → 应用 → 案例
- **步骤教程型**：固定开头 → 准备 → 步骤1-5 → 注意事项 → 总结
- **经验分享型**：固定开头 → 背景 → 踩坑 → 经验 → 建议

### 去AI味优化策略

- 开头必须是【大家好，我是快乐IT啊。】
- 加入大量个人化表达和口语化词汇
- 适当使用语气词和感叹句
- 加入真实案例和个人经历（越多越好！）
- 使用设问句增加互动感
- 调整句式节奏，避免过于规整
- 加入"快乐IT啊"的个人故事和感受

### 工作流程

```
用户需求
    ↓
选题生成（可选联网搜索热点）
    ↓
大纲设计
    ↓
模板匹配
    ↓
正文撰写（必须包含固定开头）
    ↓
去AI味优化（加入大量个人经验）
    ↓
配图建议
    ↓
SEO优化建议 + 标题A/B测试
    ↓
字数统计 + 阅读时长预估
    ↓
保存为markdown文件（自动）
    ↓
最终成品
```

## 常见问题FAQ

**Q1：Skill命名有什么要求？**
A：建议用小写字母和连字符，比如`my-awesome-skill`，不要用空格和特殊字符。

讲真，我第一次命名的时候用了空格，
结果出问题了，
后来才知道要用连字符...
哎呀，低级错误！

**Q2：description怎么写才好？**
A：记住这个公式："做什么 + 什么时候调用"。比如："写Python代码。Invoke when user asks for Python coding help."

**Q3：可以创建多个Skill吗？**
A：当然可以！每个Skill放在不同的目录里就行，想用哪个就调用哪个。

**Q4：创建好的Skill怎么用？**
A：直接和AI对话就行，AI会根据你的需求自动判断要不要调用某个Skill。也可以用`/skill`命令手动选择。
