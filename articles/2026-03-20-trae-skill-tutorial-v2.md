## 什么是Trae Skill？

先来说说什么是Skill吧。

其实很简单，Skill就像是给AI的"使用说明书"，
告诉AI在什么情况下应该做什么、怎么做。

说真的，我第一次接触Skill的时候，
还觉得这东西肯定很复杂，
结果看了文档才发现，
原来就这么简单！

比如你经常要写公众号文章，
就可以创建一个"公众号文章生成器"Skill，
以后只要说"帮我写篇公众号文章"，
AI就会自动按照你设定的流程来帮你生成。

**Skill能用来做什么？**
- 定义特定领域的工作流程
- 保存常用的提示词和模板
- 让AI更好地理解你的需求
- 打造专属的AI助手

## 准备工作

在开始创建之前，
我们先来了解一下Skill的基本结构。

**一个完整的Skill需要：**
1. 一个目录：`.trae/skills/<skill-name>/`
2. 一个主文件：`SKILL.md`（放在上面的目录里）

就这么简单！记住这两点就行。

结果发现就是写markdown文档，

## 手把手创建Skill（核心步骤）

好，我们开始！
就以我刚才创建的"微信公众号文章生成器"为例，
一步步教你怎么创建自己的Skill。

讲真，这个Skill我前前后后改了好几个版本，
踩了不少坑，
今天都分享给你们，
让你们少走弯路~

### 步骤1：确定Skill的功能和定位

首先得想清楚：
- 这个Skill是用来做什么的？
- 目标用户是谁？
- 核心功能有哪些？

比如我要做公众号文章生成器，
那核心功能就是：选题、大纲、正文、模板匹配、去AI味优化、联网搜索。

讲真，这一步我纠结了好久，
一开始想加很多功能，
后来发现贪多嚼不烂，
还是先把核心功能做好最重要。

给你们个建议：
一开始不要追求完美，
先做一个最小可用版本，
后面再慢慢迭代。

### 步骤2：创建目录结构

打开终端，执行这行命令：
```bash
mkdir -p .trae/skills/wechat-article-generator
```

或者在文件管理器里手动创建文件夹也行，
路径要对：`.trae/skills/你的skill名字/`

说实话，我第一次创建的时候，
路径写错了，
结果AI找不到Skill，
折腾了半天，
后来才发现是路径的问题...
哎呀，现在想想还觉得好笑。

### 步骤3：编写SKILL.md主文件

这是最关键的一步！
在刚创建的目录里新建一个`SKILL.md`文件。

**frontmatter部分（必须有）：**
```markdown
---
name: "wechat-article-generator"
description: "微信公众号热门文章生成器，支持选题、大纲、正文生成、模板匹配、去AI味优化和联网搜索。Invoke when user needs to generate WeChat official account articles."
---
```

注意！`description`很重要，
一定要包含：
1. 这个Skill做什么
2. 什么时候调用它

讲真，我第一次写description的时候，
只写了做什么，
没写什么时候调用，
结果AI根本不知道什么时候用这个Skill，
后来看了文档才知道要加上"Invoke when..."这部分。

**正文部分（详细说明）：**
然后在下面写详细的使用说明，
比如：
- 功能介绍
- 使用方法
- 内置模板
- 示例对话
- 注意事项

这些内容写得越详细，
AI用起来就越顺手！

### 步骤4：补充详细文档（可选但推荐）

如果你的Skill比较复杂，
可以再创建一些辅助文档，
比如我就创建了：
- `workflow.md` - 工作流程详解
- `templates.md` - 文章模板系统
- `de-ai-optimization.md` - 去AI味优化策略
- `web-search.md` - 联网搜索集成方案
- `enhanced-features.md` - 增强功能详解
- `author-persona.md` - 作者人设
- `example.md` - 完整使用示例

这些文档AI都会自动参考，
能让Skill的表现更好！

讲真，一开始我觉得没必要写这么多文档，
后来发现写得越详细，
AI的表现就越好，
花的时间还是值得的！

## 常见问题FAQ

**Q1：Skill命名有什么要求？**
A：建议用小写字母和连字符，比如`my-awesome-skill`，不要用空格和特殊字符。

讲真，我第一次命名的时候用了空格，
结果出问题了，
后来才知道要用连字符...
哎呀，低级错误！

**Q2：description怎么写才好？**
A：记住这个公式："做什么 + 什么时候调用"。比如："写Python代码。Invoke when user asks for Python coding help."

**Q3：可以创建多个Skill吗？**
A：当然可以！每个Skill放在不同的目录里就行，想用哪个就调用哪个。

**Q4：创建好的Skill怎么用？**
A：直接和AI对话就行，AI会根据你的需求自动判断要不要调用某个Skill。也可以用`/skill`命令手动选择。
