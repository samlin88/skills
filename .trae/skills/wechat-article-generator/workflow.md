# 文章生成工作流程详解

## 【重要】固定开头（必须有！）

**所有文章的第一句话必须是：**
> 【大家好，我是快乐IT啊。】

这是硬性要求，绝对不能省略！

---

## 阶段1：选题生成

### 输入要求
- 主题领域（如：编程、职场、养生等）
- 目标受众（如：初学者、进阶者、管理者等）
- 是否需要联网搜索热点

### 输出内容
- 3-5个备选选题
- 每个选题的爆款潜力分析
- 推荐选题理由

### 联网搜索集成
- 搜索近期热点话题
- 查找相关行业数据
- 搜集用户关心的问题

---

## 阶段2：大纲设计

### 输入要求
- 选定的选题
- 文章类型（干货/教程）
- 目标字数

### 输出内容
- 完整的文章结构大纲
- 每个部分的核心要点
- 建议的段落分配
- **标记出需要加入个人经验的位置**

### 大纲结构示例
```
0. 固定开场白（必须有！）
   - 【大家好，我是快乐IT啊。】

1. 开头引入（100-200字）
   - 痛点描述
   - 读者受益点
   - 【加入快乐IT啊的相关经历】

2. 主体部分（1500-2000字）
   2.1 核心概念
        【加入快乐IT啊第一次接触这个概念的经历】
   2.2 方法/步骤
        【加入快乐IT啊学习/使用这个方法的经历】
   2.3 案例/示例
        【加入快乐IT啊的真实案例】
   2.4 注意事项
        【加入快乐IT啊踩过的坑】

3. 结尾总结（100-200字）
   - 要点回顾
   - 行动号召
   - 【加入快乐IT啊的鼓励和感悟】
   - 【向读者提问，邀请留言互动】
```

## 阶段4：正文撰写

### 写作要求（重要！）

#### 硬性要求（必须满足）
1. **开头第一句必须是：【大家好，我是快乐IT啊。】**
2. **至少加入2-3个快乐IT啊的个人经历**
3. **多用"我"、"你"等人称代词**
4. **多用口语化表达**
5. **至少向读者提问1-2次**
6. **结尾邀请读者留言互动**

#### 一般要求
- 口语化表达，避免过于学术
- 每段不超过3行（手机阅读友好）
- 适当使用emoji增加生动感
- 加入小标题增强可读性

### 段落风格
```
❌ 不好的例子：
综上所述，我们可以得出结论，这个方法在大多数情况下都是有效的。

✅ 好的例子（加入个人经验）：
大家好，我是快乐IT啊。
讲真，这个方法我试过不下10次，
真的管用！
当年我就是用这个方法解决了一个大问题...
你们有没有用过类似的方法？
```

## 阶段5：去AI味优化（最重要！）

### 优化策略（按优先级排序）

#### 【最高优先级】加入大量个人经验
- 在开头加入：我第一次接触这个主题时的经历
- 在主体加入：我是怎么学习/使用的
- 在主体加入：我遇到过什么问题，怎么解决的
- 在结尾加入：我的感悟和鼓励
- 参考author-persona.md中的8大类个人经历素材

#### 【高优先级】固定开头
- **必须**在文章第一句写上：【大家好，我是快乐IT啊。】

#### 加入个人化表达
- 使用"我"、"你"等人称代词
- 加入个人经历和感受
- 使用口语化词汇

#### 调整句式节奏
- 长短句结合
- 避免过于规整的排比
- 适当使用口语化连接词

#### 增加互动感
- 使用设问句
- 向读者提问
- 邀请留言互动

#### 加入真实感细节
- 具体的数字和时间
- 真实的场景描述
- 个人化的小插曲

#### 加入"小毛病"
- 适当的语气词（哈哈、哎呀、绝了等）
- 适当的口语化表达（讲真、说实话等）
- 适当的"废话"（嗯...怎么说呢、让我想想哈等）
