# 增强功能详解

## 功能8：文件保存

### 功能说明
文章生成完成后，自动将文章保存为markdown文件到本地。

### 文件命名规则
- 格式：`articles/YYYY-MM-DD-{标题缩写}.md`
- 示例：`articles/2026-03-20-python-tutorial.md`

### 保存内容
包含完整的文章内容，以及：
- 元数据（标题、日期、字数、阅读时长）
- 配图建议
- SEO优化建议
- 版本信息

### 使用示例
```
> 用户：帮我写一篇Python入门教程
> AI：...（生成文章）...
> AI：文章已保存为：articles/2026-03-20-python-tutorial.md
```

---

## 功能9：多格式导出

### 支持格式
1. **Markdown** (.md) - 推荐，保留格式
2. **纯文本** (.txt) - 去除所有格式
3. **HTML** (.html) - 可直接预览

### 导出命令
- "导出为markdown"
- "给我一个纯文本版本"
- "生成HTML格式"

---

## 功能10：版本管理

### 版本命名
- v1.0 - 初稿
- v1.1 - 小修改
- v2.0 - 重大调整

### 文件存储
```
articles/
├── python-tutorial/
│   ├── v1.0.md
│   ├── v1.1.md
│   └── v2.0.md
└── latest.md  (指向最新版本)
```

### 版本对比
可以要求：
- "对比v1.0和v2.0的差异"
- "回退到v1.0版本"
- "基于v1.1继续修改"

## 功能11：SEO优化建议

### 优化内容

#### 1. 关键词建议
- 主关键词（1-2个）
- 长尾关键词（3-5个）
- 相关关键词

#### 2. 摘要优化
- 150-200字的摘要
- 包含主关键词
- 吸引人点击

#### 3. 标题优化
- 包含主关键词
- 20-30字最佳
- 有吸引力

#### 4. 正文优化
- 关键词密度建议
- 内部链接建议
- 图片ALT文字建议

### 输出示例
```
【SEO优化建议】

📌 主关键词：Python入门
📌 长尾关键词：Python零基础学习、Python入门教程、怎么学Python
📌 相关关键词：编程入门、Python安装、Python基础语法

📝 推荐摘要：
想学习Python但不知道从哪下手？这篇零基础入门教程，
从安装到写第一个程序，手把手教你，看完就能上手！

💡 优化建议：
- 在第一段就出现"Python入门"
- 正文中自然提及关键词3-5次
- 给图片添加ALT文字
```

## 功能12：标题A/B测试

### 生成数量
通常生成3-5个备选标题

### 评分维度
1. **吸引力**（0-10分）- 是否想点击
2. **清晰度**（0-10分）- 是否知道讲什么
3. **关键词**（0-10分）- 是否包含关键词
4. **综合评分**（0-10分）

### 输出示例
```
【标题A/B测试】

1. 《零基础学Python，这一篇就够了！保姆级教程》
   🎯 吸引力：9分
   📖 清晰度：10分
   🔑 关键词：9分（包含Python、零基础、教程）
   ⭐ 综合：9.3分
   💡 推荐理由："保姆级教程"和"这一篇就够了"对小白很有吸引力

2. 《Python到底难不难？小白看完这篇就能写代码》
   🎯 吸引力：10分
   📖 清晰度：8分
   🔑 关键词：8分
   ⭐ 综合：8.7分

3. 《从0到1学Python，我整理了这3个核心知识点》
   🎯 吸引力：7分
   📖 清晰度：9分
   🔑 关键词：10分
   ⭐ 综合：8.7分

🏆 推荐使用第1个标题！
```

## 功能13：字数统计&阅读时长

### 统计内容
- **总字数** - 包含标点和空格
- **汉字数** - 仅统计汉字
- **段落数**
- **小标题数**

### 阅读时长预估
- 普通阅读：300字/分钟
- 快速阅读：500字/分钟
- 精读：150字/分钟

### 公众号建议字数
- 最佳：1500-2500字
- 短篇：800-1500字
- 长篇：2500字以上

### 输出示例
```
【字数统计】

📊 总字数：2156字
📝 汉字数：1892字
📄 段落数：42段
🏷️ 小标题数：8个

⏱️ 阅读时长预估：
- 普通阅读：约7分钟
- 快速阅读：约4分钟
- 精读：约14分钟

💡 建议：当前字数2156字，在最佳范围内（1500-2500字），继续保持！
```

## 功能14：文章迭代

### 迭代方式

#### 1. 整体修改
- "把开头改得更吸引人"
- "让文章更口语化一些"
- "增加更多案例"

#### 2. 局部修改
- "把第二段改一下"
- "第3部分太枯燥了，重新写"
- "结尾再加个总结"

#### 3. 风格调整
- "改成更严肃的风格"
- "再活泼一点"
- "用第一人称写"

### 迭代流程
```
当前版本
    ↓
用户提出修改需求
    ↓
AI理解需求并修改
    ↓
生成新版本（v1.1）
    ↓
自动保存
    ↓
询问用户是否满意
```
