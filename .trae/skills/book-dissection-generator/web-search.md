# 联网搜索集成方案

## 搜索应用场景

### 1. 书籍信息搜索
- 搜索书名、作者、出版信息
- 查找书籍简介和目录
- 搜集书籍评价和评分
- 了解作者背景和其他作品

### 2. 拆解前准备
- 搜索书籍的核心观点
- 查找其他读者的读后感
- 搜集相关案例和故事
- 了解书籍的影响力

### 3. 拆解过程中
- 搜索书中提到的概念和理论
- 查找相关的数据支撑
- 搜集名人名言
- 了解相关背景知识

---

## 搜索关键词策略

### 书籍信息搜索关键词

| 需求 | 推荐搜索词 |
|-----|-----------|
| 基本信息 | "[书名] 作者 出版时间" |
| 内容简介 | "[书名] 内容简介 目录" |
| 读者评价 | "[书名] 读后感 评价" |
| 作者信息 | "[作者名] 简介 其他作品" |

### 拆解内容搜索关键词

| 需求 | 推荐搜索词 |
|-----|-----------|
| 核心观点 | "[书名] 核心观点 主要内容" |
| 名人评价 | "[书名] 名人推荐 评价" |
| 相关案例 | "[主题] 成功案例 真实故事" |
| 数据支撑 | "[主题] 数据统计 调研报告" |

---

## 搜索结果处理

### 信息筛选原则

1. **权威性优先** - 优先选择出版社、官方渠道、权威媒体
2. **真实性优先** - 交叉验证，避免单一来源
3. **相关性优先** - 紧扣书籍主题，避免无关信息
4. **多样性优先** - 搜集不同角度的观点和评价

### 信息整合方法

- 摘录书籍基本信息（作者、出版时间、评分等）
- 整理核心观点摘要
- 搜集有代表性的读者评价
- 用自己的语言重新组织
- 注明信息来源和时间

---

## 搜索-拆解工作流

```
用户需求
    ↓
判断是否需要搜索
    ↓
    ├─ 需要搜索 → 确定搜索关键词
    │                    ↓
    │              执行搜索
    │                    ↓
    │              筛选和整理信息
    │                    ↓
    └─────────────→ 融入拆解过程
                    ↓
                继续后续流程
```

---

## 示例：搜索辅助拆解

### 场景：拆解《原子习惯》

**步骤1：确定搜索需求**
- 需要了解这本书的基本信息（作者、出版时间、评分）
- 需要搜集其他读者的评价
- 需要查找书中的核心观点

**步骤2：设计搜索关键词**
- "原子习惯 詹姆斯·克利尔 出版时间"
- "原子习惯 豆瓣评分 评价"
- "原子习惯 核心观点 主要内容"

**步骤3：整合搜索结果**
- 提取书籍基本信息
- 搜集不同角度的读者评价
- 整理核心观点摘要

**步骤4：融入拆解文章**
- 在开头介绍书籍基本信息
- 引用有代表性的读者评价
- 用搜索到的信息支撑拆解
- 注明信息来源和时间
